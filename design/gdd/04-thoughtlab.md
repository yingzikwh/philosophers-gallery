# 04 · 思想实验与抉择 GDD（Thought Experiments & Choices）

> 机制编号：M4　|　支柱：P1 思辨即玩 / P4 抉择塑己　|　动词：抉择（Choose）
> 版本：v1.0　|　作者：文策渊

---

## 1. 概述（Overview）

沉浸式叙事玩法：玩家进入"思想实验"章节，在场景节点做抉择；每个抉择选项标注主流派/主题，由对应流派代表哲学家用其 `systemPrompt` + `coreIdeas`/`keyConcepts` 流式生成回应。玩家累积**思想倾向向量 T**（30 主题 × 35 流派），章末据 T 生成多结局，定义"思想人格"。

**复用既有**：`philosopher-chat`(mode='thoughtlab')、哲学家 `themes`/`keyConcepts`/`historicalContext`、多哲学家 persona 并排呈现。

---

## 2. 设计目标（Design Goals）

| 目标 | 量化 |
|---|---|
| G1 沉浸叙事 | 章节 3–5 节点，分支可见可追溯 |
| G2 多流派回应 | Should：单节点并排 ≥3 流派视角 |
| G3 重玩价值 | 平均章节重玩 ≥1.8（D5） |
| G4 人格表达 | 思想倾向 T → 可分享"思想人格"标签 |

---

## 3. 核心循环（Core Loop）

```
选章节(随 SL 解锁) → 读场景 → 选选项(标流派/主题)
  → 流式看到该流派哲学家回应(Should:多流派并排)
  → T 向量累加 → 下一节点
  → 章末据 T.dominant 生成结局 → 发 IN/CR(首通) + 思想人格更新
  → 换路径重玩不同结局
```

---

## 4. 规则与状态（Rules & States）

### 4.1 节点结构
- 章节：`Chapter { id, title, eraTag, nodes: Node[] }`
- 节点：`Node { id, scenarioText, options: Option[] }`
- 选项：`Option { id, text, schoolTags:string[], themeTags:string[], responderIds:string[] }`
  - `responderIds`：从同 `schoolTags` 的已收藏/全量哲学家中选代表（≥1；Should 模式并排 3）。
- 章节数：8 章，每章 3–5 节点；终章(SL40) 5 节点。

### 4.2 思想倾向向量 T
- `T` 维度 = 35 流派 + 30 主题 = 65 维，初始 0。
- 选 Option 时：`T[school] += w_opt`（默认每选项对标注流派 +1，主题 +0.5）；`w_opt` 可因选项"强/弱立场"取 1 或 0.6。
- 章末：`dominant = argmax(T)` → 映射结局池（≥4 类：理性/经验/德性/功利/存在/神秘/相对…取 Top）。

### 4.3 多流派分支回应（Should）
- 单节点默认呈现"主选项对应 1 流派"回应；开启并排后，同时生成 3 个流派视角气泡（增强探索者体验，复用多人 persona）。
- 呼应机制 P3：不同流派回应即对撞，玩家对比后更能定位自身倾向。

### 4.4 叙事闭环
- 结局文本由 `dominant` 流派代表哲学家 persona 收束（"依你一路之选，你更近于××之路"）。
- 跨章连续：终章综合 8 章 T 总和 → 生成"思想人格卡"（可分享）。

---

## 5. 数值与平衡（Numbers & Balance）

- **进入成本**：普通章免费（随 SL 解锁）；标 ★ 高级章进入耗 IN 50。
- **奖励**：完成章 + IN30；首通 + CR5（周首通总 ≤40，防 CR 通胀）；重玩仅 + IN5（鼓励但不爆资源）。
- **重玩权重**：不同路径 → 不同 `dominant` → 不同结局；8 章 × 平均 4 节点 × 3 选项 ≈ 理论上千结局组合。
- **倾向衰减**：单章内 T 不跨章累积（每章独立结局），仅终章用全量，避免长线被首章绑架。

**示例**：第一章节点 A 选"以最大幸福为准"（功利主义 +1，主题'功利' +0.5），节点 B 选"守住绝对命令"（康德义务论 +1），节点 C 选"顺应自然"（道家 +1）。T 三流派各 +1 → dominant 取决于权重与后续；若功利略高 → 结局偏"功利主义人格"。

---

## 6. 反馈与表现（Feedback & Presentation）

| 事件 | 表现 |
|---|---|
| 选项悬停 | 显示流派/主题标签与将加倾向 |
| 流式回应 | 沿用打字机；多流派并排分栏 |
| 节点完成 | 脉络小图点亮该路径 |
| 章末结局 | 全屏卡：思想人格 + 倾向雷达（65 维压缩可视） |
| 人格卡 | 可分享图（社交者） |

**UX 落点**：思想实验为 Tab；场景阅读沿用既有卡片/弹层风格；倾向雷达复用 chart 组件。

---

## 7. 风险与缓解（Risks & Mitigation）

| 风险 | 缓解 |
|---|---|
| R1 内容生成成本高 | 预置场景文本 + 仅回应流式；缓存同选项同responder结果 |
| R2 倾向被少数节点绑架 | 每节点等权 + 章末归一化；终章才用全量 |
| R3 流派回应偏离人设 | responder 严格用其 `systemPrompt`+`keyConcepts`，温度 0.3 |
| R4 重玩动力不足 | 结局集成就就 + 人格卡分享 + 周常重玩任务 |

---

## 8. 验收标准（Acceptance Criteria）

- [ ] 章节节点按 `Node/Option` 结构渲染，选项标签正确。
- [ ] 选选项后 T 向量按权重累加，章末 `dominant` 映射结局正确。
- [ ] 并排模式同节点生成 ≥3 流派回应且各用其人设。
- [ ] 高级章进入扣 IN50，首通发 CR5 幂等（周≤40）。
- [ ] 思想人格卡可分享。
- [ ] 指标 D5（平均重玩 ≥1.8）达标。
