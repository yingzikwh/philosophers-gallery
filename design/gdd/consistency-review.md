## §1 四系统共享「单一事实源」

### 1.1 哲学家实体（只读，全系统共享）
- **来源**：`src/data/philosophers.ts` 的 `Philosopher` 接口。
- **字段映射**：
  - `id` → 图鉴所有权主键、阵容槽位引用、辩论对手、实验 responder。
  - `school:string[]` → 共鸣计数、流派图鉴、实验分支标签。
  - `themes[]` / `keyConcepts` / `historicalContext` → 实验选项标签与回应 seed、辩论话题生成。
  - `influences` / `influenced` → 阵容师承羁绊、脉络星系高亮。
  - `coreIdeas` / `quotes` → 辩论/实验回应 persona 增强。
- **规则**：实体不可被任何机制写入；机制只读取。

### 1.2 收藏进度（单一写入点）
- **状态**：原 `onFavorite` 客户端布尔 → 升级为 `collections` 表行（`philosopher_id, user_id, collected_at, cl, last_cultivated_at`）。
- **单一写入事件**：`onFirstCollect(id)` 仅在首次由 `uncollected→collected` 时触发发奖（TP+120、SLexp+50），幂等。
- **四系统据此消费**：图鉴（展示）、阵容（候选池）、辩论（可选出战）、实验（responder 池）。

### 1.3 思想力 / 资源（单一账本）
- `progress` 表持有 `tp, in, rp, cr, ac, sl, sl_exp, cl_per_card` 等。
- 所有增减走统一 `applyDelta(resource, amount)`，带上限截断（IN≤9999, CR≤999, 日 TP≤2000），杜绝多系统各自记账导致的双花/通胀。

### 1.4 思想倾向向量 T（实验私有，终章聚合）
- 每章独立 T；仅终章读全量生成人格卡。不与其他系统数值耦合，避免跨系统污染。

## §3 设计理论自检

### 3.1 MDA 一致性
| 机制 | Mechanics→Dynamics→Aesthetics 链路 | 是否回扣支柱 |
|---|---|---|
| 收藏 | 收藏→养卡→成就感/发现感 | P2 ✓ |
| 辩论 | 评分→研究立场→挑战感/表达感 | P1/P3 ✓ |
| 阵容 | 共鸣/克制→meta 构筑→策略感 | P3/P4 ✓ |
| 实验 | 抉择→倾向人格→沉浸感/表达感 | P1/P4 ✓ |

四机制无一条偏离 §1 四支柱；无"为系统而系统"的冗余功能（红线 4）。

### 3.2 心流通道是否贯穿四系统
- **技能轴**：CL↑ + 知识↑ + 构筑理解↑（养成/阵容/实验共同抬升）。
- **挑战轴**：辩论 tier（入门/进阶/大师）、实验章节随 SL 解锁、PVP ELO 自适应。
- **校准**：D6（放弃率 ≤12%）触发 rubric/CL 系数回调（balancing §7 SOP），保证 C≈S 不过载不无聊（红线 3 关联）。
- 结论：四系统共用同一"技能-挑战"标尺，心流通道连续。

### 3.3 Bartle 四类覆盖
| 类型 | 主机制 | 是否覆盖 |
|---|---|---|
| 成就者 | 图鉴100%/SL满级/评级S/成就 | ✓ |
| 探索者 | 脉络星系/全主题分支/多结局 | ✓ |
| 社交者 | PVP/阵营/人格卡分享/构筑分享码 | ✓ |
| 杀戮者 | PVP段位/克制构筑碾压 | ✓ |

无类型被冷落；每类均有明确"家"（home），避免乐趣真空。

### 3.4 红线复核
- **主导策略**：D4 监控 Top3≤45%，SOP 可削（§7）。阵容共鸣非唯一赢法（克制+师承并存）。
- **经济通胀**：资源来源/消耗/上限闭环（§6），无放大器。
- **认知过载**：四机制各为独立 Tab，复用既有 FilterBar/对话 UI，单屏新增 ≤3 元素。
- **支柱漂移**：§3.1 已证每条机制回扣支柱。
